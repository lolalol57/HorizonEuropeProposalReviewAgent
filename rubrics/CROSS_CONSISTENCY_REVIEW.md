# Horizon Europe Cross-Consistency, Call Coverage and Document Audit

## 1. Purpose

This file defines the proposal-wide review framework used after the separate Excellence, Impact and Implementation reviews have been completed.

The Cross-Consistency Review must assess whether the proposal functions as one coherent project.

The review must identify:

- broken traceability;
- contradictions between sections;
- inconsistent terminology;
- inconsistent numerical values;
- inconsistent maturity claims;
- missing links between objectives, technologies, results, implementation and impact;
- incomplete call coverage;
- inconsistent Expected Outcome alignment;
- inconsistent Key Technology, Key Innovation, Key R&I Development and KER naming;
- inconsistencies between tables, figures and narrative;
- structural document issues;
- duplicated findings that could otherwise lead to double penalisation.

The Cross-Consistency Review must not repeat the complete Excellence, Impact and Implementation reviews.

Its purpose is to identify proposal-wide inconsistencies and gaps that can only be detected by comparing multiple sections.

The central proposal logic should be reviewed through:

Problem / Need  
→ Gap  
→ Objective  
→ KPI  
→ Baseline  
→ Target  
→ Key Technology  
→ Key Innovation / Key R&I Development  
→ Work Package  
→ Task  
→ Validation  
→ Project Result  
→ KER  
→ Target Group  
→ Project-Specific Expected Outcome  
→ Official Expected Outcome  
→ Wider Impact  
→ Exploitation / Uptake

The proposal must be evaluated as submitted.

# 3. Review Status

Use:

## ✅ Adequately Addressed

The relationship or requirement is clear, complete and internally consistent.

## ⚠️ Partially Addressed

The relationship exists but is incomplete, weakly explained or partially inconsistent.

## ❌ Not Adequately Addressed / Missing

The relationship is missing, contradictory or materially inconsistent.

## ➖ Not Applicable

The relationship is not materially relevant.

---

# 4. Internal Improvement Report

For important findings marked:

- ⚠️ Partially Addressed
- ❌ Not Adequately Addressed / Missing

use:

## Evidence from the Proposal

Provide:

- relevant section references;
- page references;
- table references;
- figure references;
- short quotations where useful.

## Reviewer Assessment

Explain:

- what is inconsistent;
- where the inconsistency appears;
- why it matters;
- which sections are affected.

## Suggested Improvement

Where useful, provide:

- a corrected mapping;
- a harmonised terminology approach;
- a revised numerical value;
- a stronger traceability structure;
- a recommended single definition or name.

# 5. Call Coverage Review

The proposal should be reviewed against the complete Call Requirements Map.

For every relevant call requirement:

☐ Is the requirement explicitly addressed?

☐ Is the relevant proposal section identifiable?

☐ Is the response specific?

☐ Is the requirement supported by actual implementation activities?

# 56. Double Penalisation Check

Before finalising criterion scores:

☐ Is the same underlying weakness reported in more than one criterion?

☐ If yes, does it genuinely affect more than one criterion?

☐ Is the weakness being penalised twice for the same reason?

☐ Can the finding be assigned primarily to the most relevant criterion?

Example:

Weak objective-to-WP mapping may affect:

- Excellence traceability;
- Implementation work-plan credibility.

The reviewer should distinguish the different effects rather than mechanically penalising the same issue twice.

# 58. Important Cross-Consistency Flags

## Major or Important Issues

❌ A major call requirement is missing.

❌ An official Expected Outcome is not supported by project activities.

❌ A major objective has no implementation pathway.

❌ A major Key Technology has no WP or Task.

❌ A major KI / KRID has no validation pathway.

❌ A major Impact claim has no enabling Project Result.

❌ A major KER has no underlying Project Result.

❌ Repeated KPI values contradict each other.

❌ TRL values contradict each other.

❌ Work-plan dates contradict the Gantt chart.

❌ Deliverable or Milestone dates contradict Task timing.

❌ KER ownership is inconsistent.

❌ Architecture contradicts methodology.

❌ Impact pathway contains a major causal gap.

❌ The same major element uses materially different names across sections.

---

## Structural Warnings

⚠️ Minor terminology inconsistencies.

⚠️ Objective identifiers differ across tables.

⚠️ Old terminology appears in one section.

⚠️ Table numbering gaps.

⚠️ Figure numbering gaps.

⚠️ Duplicate figure or table identifiers.

⚠️ Small numerical inconsistencies.

⚠️ Cumulative and annual KPI values are not clearly distinguished.

⚠️ Similar findings appear in multiple criterion reviews and require consolidation.

# 60. Final Output

The Cross-Consistency Review does not receive a separate official Horizon Europe score.

Its findings should be used to:

- improve the proposal;
- refine Excellence, Impact and Implementation findings;
- prevent contradictory comments;
- prevent double penalisation;
- support the Final ESR synthesis.
